# __AGENT_NAME__

Standalone digital worker workspace generated by `G-Agent Creator`.

## Stack

- Flutter shell: `apps/shell`
- Node/TypeScript core: `apps/core`
- Firebase Auth, Firestore, Storage
- Cloud Run for hosted web-mode execution

## What It Is

This generated product is meant to be a general computer-work agent:

- coding and debugging
- research and synthesis
- writing and editing
- operations and repetitive workflows
- file and workspace management
- autonomous multi-step task execution

## Run Locally

```powershell
npm install
cd apps/shell
flutter pub get
cd ..
cd core
copy .env.example .env
npm install
npm run dev
```

In another terminal:

```powershell
cd apps/shell
flutter run -d windows --dart-define=DEFAULT_BACKEND_URL=http://127.0.0.1:__BACKEND_PORT__
```

## Run The Shell On Web

```powershell
cd apps/shell
flutter run -d chrome --dart-define=DEFAULT_BACKEND_URL=__CLOUD_RUN_URL__
```

## Repo Layout

- `AGENTS.md`, `SOUL.md`, `USER.md`, `memory/`: persistent digital-worker identity and continuity
- `apps/shell`: desktop + web shell
- `apps/core`: agent runtime, tools, session APIs, streaming APIs
- `agents/`: built-in role definitions for planner/executor/reviewer style workers
- `plugins/`: local runtime plugins with hook and tool registration
- `skills/`: project-local skills that the product can grow into using later
- `infra/firebase`: Firebase Hosting + rules
- `infra/cloud-run`: Cloud Run deployment notes
- `infra/openshell`: optional hardened execution policy files and setup notes

## Product Defaults

- local desktop mode can launch the core locally
- web mode talks to Cloud Run
- Firestore stores sessions/messages when Firebase admin is configured
- large tool outputs become persisted artifacts
- local runtime plugins load automatically at startup
- built-in agent roles load from `agents/*.json`
- self-evolution is enabled by default and writes learned skills under `skills/learned/`
- environment bootstrap snapshots reduce wasted discovery turns
- strict auth is enabled by default for hosted mode
- security policy is deny-by-default for origins, outbound access, and remote execution
- OpenShell is optional, but supported as a hardened execution backend
- plugin manifests are validated before runtime loading
- context assembly runs through a selected context engine
- doctor diagnostics report plugin, security, runtime, and context-engine findings

## Frontier Defaults

The generated app also includes:

- memory note APIs
- session search APIs
- automation scheduler APIs
- provider/model routing APIs
- MCP import and approval APIs
- an evaluation harness in `eval/`
- learned-skill listing and deletion APIs
- a richer built-in plugin pack for research, writing, analysis, and workspace curation
